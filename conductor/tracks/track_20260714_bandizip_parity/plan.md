## 1. 기획 배경

v0.3.0에서 압축/해제/암호의 기본기는 확보했으나, 반디집 사용자가 당연하게 기대하는
일상 기능(레거시 한글 zip 열기, 내부 파일 바로 열기, 아카이브 편집 등)이 빠져 있다.

## 2. 문제 정의

- 알집/구형 도구가 만든 cp949 파일명 zip을 열면 한글이 깨진다(한국 사용자 치명적).
- 아카이브 안의 문서/텍스트를 보려면 전체를 풀어야 한다(이미지만 뷰어 지원).
- 이미 만든 아카이브에 파일을 추가/삭제할 수 없다.
- .gz/.bz2/.xz 단일 파일 압축을 풀 수 없다.
- 우클릭 메뉴에 "열기", "각각 압축하기"가 없다.

## 3. 목표

반디집의 "기본" 사용 시나리오를 PackNine만으로 완결할 수 있게 한다.

## 4. 범위 (In/Out)

**In (이번 트랙)**
1. ZIP 파일명 코드페이지 자동 감지: UTF-8 플래그 없는 엔트리를 UTF-8 → cp949 순으로
   판별(반디집 "코드 페이지 자동 감지"에 대응)
2. 단일 파일 압축 해제: .gz / .bz2 / .xz (tar가 아닌 순수 단일 파일)
3. 아카이브 내부 파일 더블클릭 → 임시 해제 후 기본 연결 프로그램으로 열기
   (이미지는 기존 내장 뷰어 유지)
4. 탐색기 우클릭 메뉴 확장: 아카이브 "PackNine으로 열기", 다중 선택 "각각 압축하기"
5. 아카이브 편집: GUI에서 파일 추가/선택 삭제 (전 포맷 공통 재작성 방식, 원자적 교체)
6. GUI 파일 목록 칼럼 정렬

**Out (후속 트랙 후보)**
- 분할 압축(볼륨) 생성/해제 — zipfile 미지원, 자체 포맷 처리 필요
- EGG/ALZ 해제 — 신뢰할 수 있는 파이썬 구현 부재, 별도 조사 필요
- 탐색기식 폴더 트리 UI, 압축 미리보기, 코드 서명

## 5. 사용자 시나리오

- 오래된 알집 zip을 더블클릭하면 한글 파일명이 그대로 보이고 정상 해제된다.
- 아카이브에서 보고서.pdf를 더블클릭하면 PDF 뷰어가 바로 뜬다.
- 열려 있는 zip에 파일을 추가하거나 잘못 들어간 파일을 지운다.
- 파일 5개를 선택해 우클릭 → "각각 압축하기" → 파일별 zip 5개가 생긴다.

## 6. 성공 기준

- cp949 zip 라운드트립 테스트 통과, 기존 UTF-8 zip 회귀 없음
- 신규 기능 전부 TDD 테스트 보유, 전체 스위트 통과
- 편집 실패 시 원본 아카이브 무손상(원자적 교체)

## 7. 리스크 및 가정

- zipfile 내부 인코딩 동작(cp437 폴백)에 의존 → 표준 라이브러리 동작이라 안정적
- 단일 파일 gz는 원본 크기를 신뢰할 수 없어(ISIZE 32bit) 해제 시 스트리밍 압축비
  상한으로 압축폭탄을 방어한다
- 아카이브 편집은 재작성 방식이라 대용량에서 느릴 수 있음(정확성 우선, 성능은 후속)
